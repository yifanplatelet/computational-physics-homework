\section*{Problem 5}

Consider the sequence
\[
Z_2 = 2,
\]
\[
Z_{n+1} = 2^{\,n-\frac12}\sqrt{1-\sqrt{1-4^{1-n}Z_n^2}},
\qquad n=2,3,\dots
\]
The goal is to compute this sequence numerically, compare the result with the exact limit
\[
Z_n \to \pi \quad (n\to\infty),
\]
and explain why severe rounding-error propagation appears when $n \ge 30$.

In this report, the original unstable recursion is kept for comparison, while a numerically stable reformulation is used as the reference method. The document also explains why the first error model failed and how the corrected algorithm avoids the same cancellation in both the main recursion and the error analysis.

#include <stdio.h>
#include <stdlib.h>
#include <math.h>
#include <float.h>

#ifndef M_PI
#define M_PI 3.14159265358979323846264338327950288L
#endif

/* --------------------------------------------------
 * Original recursion: intentionally kept to show the
 * loss of accuracy caused by cancellation.
 * -------------------------------------------------- */
void compute_naive(int N, double *z) {
    for (int i = 0; i <= N; ++i) z[i] = 0.0;
    z[2] = 2.0;

    for (int n = 2; n < N; ++n) {
        double a = ldexp(z[n] * z[n], 2 - 2 * n);
        double inner = 1.0 - sqrt(1.0 - a);
        z[n + 1] = pow(2.0, n - 0.5) * sqrt(inner);
    }
}

/* --------------------------------------------------
 * Stable recursion:
 * 1 - sqrt(1-a) = a / (1 + sqrt(1-a))
 * Hence
 * z_{n+1} = sqrt(2) * z_n / sqrt(1 + sqrt(1-a_n)).
 * -------------------------------------------------- */
void compute_stable(int N, long double *z) {
    for (int i = 0; i <= N; ++i) z[i] = 0.0L;
    z[2] = 2.0L;

    for (int n = 2; n < N; ++n) {
        long double a = ldexpl(z[n] * z[n], 2 - 2 * n);
        long double s = sqrtl(1.0L - a);
        z[n + 1] = sqrtl(2.0L) * z[n] / sqrtl(1.0L + s);
    }
}

/* Stable evaluation of t = 1 - sqrt(1-a). */
long double stable_t(long double a) {
    long double s = sqrtl(1.0L - a);
    return a / (1.0L + s);
}

int main(void) {
    int N;
    printf("Enter N (recommended 40~60): ");
    if (scanf("%d", &N) != 1 || N < 2) {
        printf("Invalid input.\n");

    long double *z_stable = (long double *)malloc((N + 1) * sizeof(long double));

        printf("Memory allocation failed.\n");
        free(z_naive);
        free(z_stable);
        return 1;

\section{Theoretical Analysis}

Let
\[
a_n = 4^{1-n} Z_n^2.
\]
Then the original recursion can be written as
\[
Z_{n+1} = 2^{n-\frac12}\sqrt{1-\sqrt{1-a_n}}.
\]

The dangerous part is
\[
1-\sqrt{1-a_n}.
\]
When $n$ is large, $a_n$ is very small, so
\[
\sqrt{1-a_n} \approx 1 - \frac{a_n}{2},
\]
and therefore
\[
1-\sqrt{1-a_n} \approx \frac{a_n}{2}.
\]
This means the program subtracts two numbers that are both extremely close to $1$, which causes catastrophic cancellation and destroys many significant digits.

To remove this instability, rationalize the expression:
\[
1-\sqrt{1-a_n}
= \frac{(1-\sqrt{1-a_n})(1+\sqrt{1-a_n})}{1+\sqrt{1-a_n}}
= \frac{a_n}{1+\sqrt{1-a_n}}.
\]
Substituting this back into the recursion gives
\[
Z_{n+1}
= \frac{\sqrt{2}\,Z_n}{\sqrt{1+\sqrt{1-a_n}}},
\]
which is numerically stable.

For the rounding-error analysis, define
\[
t_n = 1 - \sqrt{1-a_n}.
\]
Again, $t_n$ must \emph{not} be evaluated by direct subtraction. It must be computed as
\[
t_n = \frac{a_n}{1+\sqrt{1-a_n}}.
\]
Otherwise, the error model itself suffers from the same cancellation and becomes useless in the large-$n$ regime.

The local relative error of the unstable subtraction is approximately
\[
\frac{|\delta t_n|}{|t_n|} \approx \frac{\varepsilon}{t_n},
\]
where $\varepsilon$ is the machine precision of \texttt{double}. Since the outer square root halves relative errors to first order, the induced relative error in $Z_{n+1}$ is approximately
\[
\frac{|\delta Z_{n+1}|}{|Z_{n+1}|} \approx \frac{1}{2}\frac{\varepsilon}{t_n}.
\]
Because $t_n \approx a_n/2$, this becomes
\[
\frac{|\delta Z_{n+1}|}{|Z_{n+1}|} \approx \frac{\varepsilon}{a_n}.
\]
Using the fact that $Z_n \to \pi$, we obtain
\[
a_n = 4^{1-n} Z_n^2 \approx \frac{\pi^2}{4^{n-1}},
\]
and therefore
\[
\boxed{
\frac{|\delta Z_{n+1}|}{|Z_{n+1}|}
\approx
\frac{\varepsilon 4^{n-1}}{\pi^2}
}.
\]
This estimate explains why the error grows slowly at first but then increases dramatically when $n$ reaches roughly $30$.

\section{Interpretation of the Numerical Results}

The computation contains two separate tracks:
\begin{enumerate}[label=(\arabic*)]
    \item the \textbf{naive recursion}, which follows the original formula exactly;
    \item the \textbf{stable recursion}, which is algebraically equivalent but avoids subtracting nearly equal numbers.
\end{enumerate}

The stable recursion is used as the numerical reference. In the output,
\begin{itemize}
    \item \texttt{|naive-pi|} shows how the direct implementation drifts away from $\pi$;
    \item \texttt{|stable-pi|} stays small and confirms the correct limiting value;
    \item \texttt{real\_abs\_err} measures the actual difference between the unstable and stable algorithms;
    \item \texttt{rel\_local\_est} and \texttt{log10(rel\_est)} show the size and growth trend of the local cancellation error;
    \item the final table compares the measured local estimate with the asymptotic formula $\varepsilon 4^{n-1}/\pi^2$.
\end{itemize}

A key correction relative to the failed version is that the error-analysis code no longer computes
\[
1 - \sqrt{1-a_n}
\]
by direct subtraction. That was the reason the previous implementation produced zeros or unrealistic values in the later rows. The corrected program evaluates the same quantity with the rationalized form, so the analysis remains stable even when the original recursion has already broken down.

\section{Final Conclusion}

This problem is not difficult from the theoretical point of view: the sequence converges to $\pi$. The real challenge is numerical stability. The original formula contains the term
\[
1 - \sqrt{1-a_n},
\]
which becomes numerically dangerous once $a_n$ is very small. As a result, direct floating-point evaluation loses significant digits and the error is amplified in later steps.

The correct fix is not merely to compute the sequence with higher precision, but to rewrite the formula into a stable form and to use the same stable form in the error estimator. After this correction, the program clearly demonstrates three facts:
\begin{enumerate}[label=(\arabic*)]

    \[

\end{enumerate}
